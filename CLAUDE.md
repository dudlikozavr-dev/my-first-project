# Правила работы с проектом

## Структура файлов

- Каждая страница — один самостоятельный HTML-файл
- Никаких внешних библиотек, фреймворков и CDN-ссылок
- CSS — внутри `<style>` в том же файле
- JS — внутри `<script>` в том же файле
- Иконки — только inline SVG, без иконочных шрифтов

## Дизайн

- Фон: тёмный (`#0d0a0f` или близкий)
- Текст: светлый, белый или с небольшой прозрачностью
- Акцентный цвет: оранжевый (например `#f59e0b`, `#d97706`)
- Допустимы градиенты с переходом оранжевый → розовый → золотой
- Стиль карточек: glassmorphism (полупрозрачный фон + размытие)
- Анимации: плавные, ненавязчивые (scroll-reveal, hover, pulse)

## Адаптивность

- Все страницы корректно отображаются на мобильных (от 320px) и десктопе
- Использовать `clamp()` для размеров шрифтов
- Сетки через CSS Grid или Flexbox с `auto-fit` / `auto-fill`
- Медиазапросы при необходимости — через `@media(max-width: ...)`

## Текст на страницах

- Простой и дружеский язык — без канцелярита и официоза
- Короткие предложения, живые формулировки
- Обращение на «ты» к покупателю

## Рабочий процесс

- Перед созданием нового файла — сначала описать план: что будет на странице, из каких блоков состоит, какие данные нужны
- Только после подтверждения — приступать к коду
- Коммиты делать с понятными сообщениями на русском языке
